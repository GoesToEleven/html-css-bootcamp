• svg is XML
    • svg is XML describing an image
    • XML is case sensitive.
    • You must close all tags.
    • Attributes must be inside quotes.
    • XML header lines
      • Example:
        • <?xml version="1.0" encoding="utf-8"?>
      • should be removed when you are embedding an SVG in an HTML page or another SVG
      • Well-formed XML documents contain only one prolog. So they should be removed if you are embedding one SVG inside another. See: this for more info
      • When embedding an SVG inside an HTML file, you could keep the prolog lines. The HTML parser in the browser will ignore them. But given they serve no purpose, it is my advice that you "should" remove them to make your document smaller - and more readable. :)
  • The <svg> element is a root element.
  • A Doctype Declaration (DTD) should be left off - leads to more problems than it solves.
    • Don't use this: 
      • <!DOCTYPE svg PUBLIC "-//W3C//DTD SVG 1.1//EN"
"http://www.w3.org/Graphics/SVG/1.1/DTD/svg11.dtd">
    • Use this: 
      • <svg version="1.1"
     baseProfile="full"
     xmlns="http://www.w3.org/2000/svg"
     xmlns:xlink="http://www.w3.org/1999/xlink"
     xmlns:ev="http://www.w3.org/2001/xml-events">
      • "version" and "baseProfile" identify the version of the SVG and should always be used instead of DTD.
      • the last two lines cause errors in webstorm for me so I leave them off with slight reservation
  • The order of rendering of elements matters - later elements are rendered atop previous elements.
  • CSS
    • Avoid the inline 'style' attribute where possible
      • Not This 
        • <circle style="fill:red; stroke:blue;" ... />
      • But This 
        • <circle fill="red" stroke="blue" ... />
  • Specify units when assigning lengths to properties
    • The only way to avoid problems is to always specify a unit when assigning lengths to properties.
    • Happily, in SVG, px units are defined to be equivalent to the units established by the current coordinate system (user units).
    • In other words, wherever you would otherwise have omitted the unit from a length assigned to a property, use the px unit instead.
    • For example, instead of writing: 
      • <text stroke-width="2" style="font-size:20;" ...>
    • write: 
      • <text stroke-width="2px" style="font-size:20px;" ...>
    • According to the SVG 1.1 Property Index, there are only eight properties applicable to SVG 1.1 that accept a length value - it is these properties that you should be careful to always specify a length for:
      • stroke-width
      • stroke-dashoffset
      • font
      • font-size
      • baseline-shift
      • kerning
      • letter-spacing
      • word-spacing

    • MDN does not include px units in the examples that follow. I have not added them to MDN's examples. MDN's example are more recent than jwatt's article. Perhaps this isn't needed anymore? Look to see if the code breaks anywhere to make the determination.
